Quasar Alarm

The Rationale:

Create an alarm that encourages a person to get out of bed, rather than punishing a person if they don't.

Make the alarm simple and easy to use.

Easy to extend, make it so any other person who knows Js & HTML can easily add new functionality.

Make it dependable.

The Solution:

Allow users to define different actions that can be performed when an alarm goes off. Perhaps they want to listen to everything that is on their to do list for the day, what the weather is like, and then listen to a live stream of NPR. Instead of being pestered to wake up, they are prepared for the day ahead.

New Users

The first thing they are taken to is an empty alarm. They see the time, the alarm name, and a button that says 'Add'. When the add button is clicked, the user will be given a list of default actions with a '+' next to each one. Clicking the '+' tells the user that they have added the action to the alarm. Clicking the dropdown shows the various settings, perhaps an area code if they tap the Weather action. Here they can also test the alarm to see what it does.

Creating a new procedure

The ability to create a new action is available from the same menu that is displayed when a user tries to add an alarm, consolidating it in a single place.

Create a new alarm

At the bottom is a row of 4 circles which designate different alarms. Each alarm can be customized to only go off on certain days and be given a title.

Vue & Quasar

I used the quasar framework, a web framework for creating web, mobile, and desktop applications powered by Vue 2. Quasar provides a material theme and useful vue components that make an application feel and control like a native android app, it's also easy to customize.

Testing

Through user testing and interviews I was able to discern what functionality would be most beneficial to add.

  • Alerting the user when they open the alarm that the audio volume is low.
  • Displaying the amount of time until the alarm will ring.
  • Feedback for every action, specifically when a new procedure has been added to the alarm.
  • Quickly adding and editing new procedures. The major change from my wireframes was converting created procedures to dropdowns.
  • Be able to test newly created actions.
